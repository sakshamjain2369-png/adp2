Project Overview

"Saksham's Reading Room" is a desktop library manager. The frontend (Tkinter) offers a modern sidebar + table layout; the backend handles search, filter, sort, add, delete, and update; storage persists everything to a JSON file so changes survive restarts.

Frontend
Tkinter treeview UI with search, filters, sort, inline add/edit/delete.
Backend
BookManager orchestrates filtering, searching, sorting, CRUD, and stats.
Storage
BookStorage reads/writes JSON (`media.json`) with project-root resolution.
Data
Fields: name, author, date (year), category (Novel/Philosophy/Poetry).

Architecture

Backend · `BookManager`

Located at `backend/book_manager.py`. Owns business logic:

  • Get all books; filter by category; search by name (case-insensitive).
  • Sort by year (newest-first or oldest-first).
  • Add, delete, update books with immediate persistence.
  • Compute counts for categories: total, novels, philosophy, poetry.

Frontend · `ModernLibraryGUI`

Located at `frontend/gui.py`. Tkinter UI with:

  • Sidebar navigation for All, Novels, Philosophy, Poetry.
  • Search-as-you-type input; category + sort controls.
  • Treeview table for Title/Author/Year/Category; right-click menu.
  • Inline add/edit/delete form plus modal add dialog.
  • Status + counters; save button to persist quickly.

Storage · `BookStorage`

Located at `storage/storage.py`. Handles persistence:

  • Resolves relative paths to project root (`media.json`).
  • Load books from JSON into memory.
  • Save books back to JSON with indentation.
  • Expose `get_books` / `set_books` for managers/UI.

Data Model

Books live in `media.json` as a JSON array of objects.

Feature Highlights

Search & Filter

Instant search on title; quick filters by category; combined with sort toggles.

Sorting

Newest-first or oldest-first based on year; integrated with category filters.

Inline CRUD

Add, edit, delete directly from the inline form or via context menu; persists immediately.

Persistent JSON

All changes saved to `media.json`; safe for small to medium personal collections.

Tests

Pytest-based coverage across storage, backend, and frontend.

Storage

`tests/test_storage.py` — ensures JSON load/save and year ordering logic work as expected.

Backend

`tests/test_book_manager.py` — verifies add and retrieval flows.

Frontend

`tests/test_gui.py` — instantiates the Tkinter UI and asserts book rows render in the treeview.

Run: pytest tests/ -v
Requires: Python 3.7+, Tkinter (standard library)

How to Run

  1. Install Python 3.7+ (Tkinter included on most installs).
  2. From project root: python online_book_project/main.py
  3. Library data lives in online_book_project/media.json. Keep it alongside main.py.
Tip: Back up media.json before large edits. Avoid manual edits while the app is running.

Extend & Improve

Ideas

  • Add cover images and display thumbnails in a sidebar.
  • CSV/Excel import-export for bulk management.
  • SQLite backend for larger catalogs and multi-user use.
  • Ratings, reading history, and custom tags.

Where to start

  • `frontend/gui.py` for UI adjustments and new controls.
  • `backend/book_manager.py` for new filters/sorts/analytics.
  • `storage/storage.py` to evolve data format or add migrations.
  • `tests/` to keep quality in check as features grow.
